# Investigation: Sprite download cancelled for wd_collectibles_v2.png
## TLDR
This is a moderate-severity client error: a remote sprite download is being cancelled/timeouted, and the failure is surfaced as an exception instead of a soft fallback. The UnityWebRequest in `DownloadOrLocalPics` is cancelled, leaving no cached file, and callers bubble it up as “Failed to download or find” for `wd_collectibles_v2.png`.

## Risk Assessment
Moderate: users can see missing UI art and repeated error logging; likely non-crashing but noisy and can degrade UX.

## Release Branch
v3.34.4

## Release Version
3.30.15

## Error Summary
`System.OperationCanceledException` from UnityWebRequest occurs while downloading `wd_collectibles_v2.png`. The download is cancelled (likely timeout or lifecycle cancellation), the file is missing in persistent data, and the caller raises a generic “Failed to download or find” exception.

## Root Cause
`DownloadOrLocalPics.TryDownloadAndCacheAsync` uses a linked cancellation token with a timeout, and the request is aborted on cancellation. When this happens, the method returns `null`, leaving no cached file. A caller expects the file to exist and throws an exception instead of treating cancellation/missing content as a soft failure. The issue is amplified if the CDN asset is slow/unreachable or the asset key is not present locally.

## Affected Code
- `Assets/Scripts/Utils/DownloadOrLocalPics.cs` (TryDownloadAndCacheAsync, DownloadOrGetLocalPath, GetSpriteAsync)

## Suggested Fix
- Treat cancellation/timeout as a soft failure: ensure callers do **not** throw when `GetSpriteAsync` returns default; log a warning and fall back to a placeholder/local sprite instead.
- Optionally add a retry with backoff for timeouts and/or increase `_timeout` for large assets.
- Verify `wd_collectibles_v2.png` exists in the CDN or in the local sprites provider; if not, add it or fix the asset key.
