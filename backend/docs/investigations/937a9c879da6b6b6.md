# Investigation: Coupons service not initialized during live op presentation

## TLDR
Medium severity race condition: when a coupon live op times out while a preceding popup (e.g., PersonalOffers) is still being displayed, the coupon service gets reset via `RemoveLiveOp`, but the coupon remains in the presentation queue. When the preceding popup closes and the coupon's turn arrives, `HandleLiveOpsSelectedAsync` finds `_dynamicInAppOperation == null` and logs an error. The error is non-blocking (the coupon is skipped and presentation continues), but it affects 174 users and fires consistently for live op `699ffce40900377994a8fb42`.

## Risk Assessment
**Medium** — The error does not crash the app or block the user flow. `HandleLiveOpsSelectedAsync` returns `default`, causing `ShopCouponPresenter` to call `showNextCallback`, so the next live op in the queue is presented normally. However, the user misses seeing the coupon popup they were entitled to, which has a direct revenue impact (coupon activation drives purchases). Affects both payers and non-payers across iOS and Android.

## Root Cause
**Race condition between live op timeout removal and async presentation queue.**

The flow that triggers the error:

1. `LiveOpsService.InitializeLiveops` runs for a trigger (e.g., `ENTER_LOBBY`). It finds multiple matching live ops including a coupon and a PersonalOffers popup.
2. For the coupon, `TryRegisterController` → `ShopCouponService.RegisterLiveOp` succeeds, setting `_dynamicInAppOperation`. The coupon is added to `liveopsForShow`.
3. `RemoveAfterTimeout(timeLeft, couponOp).Forget()` is started for the coupon (`LiveOpsService.cs` ~line 689).
4. `PresentLiveOps` starts iterating `liveopsForShow`. PersonalOffers is shown first (based on placement ordering).
5. **While the user is viewing the PersonalOffers popup**, the coupon's `timeLeft` expires.
6. `RemoveAfterTimeout` fires → `RemoveLiveOp` → `UnregisterLiveOp` → `ShopCouponService.Remove` → `Reset()` → **`_dynamicInAppOperation = null`**, and `_liveOpsToEndTime.Remove(couponOp.Id)`.
7. User closes PersonalOffers → `OnShowNext` callback fires → `LiveOpsPresenter.IterateDynamicInApps` moves to the coupon.
8. `LiveOpsToEndTime.TryGetValue(couponId)` returns `false` (entry was removed in step 6) → `timeLeft = null`.
9. `ShopCouponPresenter.HandleShowLiveOpAsync` → `ShopCouponService.HandleLiveOpsSelectedAsync` → `_dynamicInAppOperation == null` → **error logged**.
10. Returns `default` (`SkipNextLiveOps = false`) → `showNextCallback` is called → presentation continues normally.

The "Time out: False" in the error message is explained by `timeLeft` being `null` (the entry was removed from `LiveOpsToEndTime`), so `timeLeft != null && timeLeft.Value <= TimeSpan.Zero` evaluates to `False`.

## Affected Code
- `Assets/Scripts/Coupons/ShopCouponService.cs` — `HandleLiveOpsSelectedAsync` (error origin, ~line 185)
- `Assets/Scripts/Coupons/ShopCouponPresenter.cs` — `HandleShowLiveOpAsync` (caller)
- `Assets/Scripts/LiveOps/LiveOpsPresenter.cs` — `IterateDynamicInApps` (presentation queue that doesn't validate live op validity before presenting)
- `Assets/Scripts/LiveOps/LiveOpsService.cs` — `RemoveAfterTimeout` / `RemoveLiveOp` (timeout removal that doesn't account for in-progress presentation)

## Suggested Fix

**Option A (Recommended) — Guard in `LiveOpsPresenter.IterateDynamicInApps`:**

Add a validity check before presenting each live op. If the live op has been removed (e.g., timed out) while a previous popup was displayed, skip it:

```csharp
// In LiveOpsPresenter.IterateDynamicInApps, after MoveNext() and before calling HandleShowLiveOpAsync:
if (dynamicInAppEnumerator.MoveNext())
{
    var dynamicInAppOperation = dynamicInAppEnumerator.Current;

    // Skip live ops that were removed while a previous popup was displayed
    if (!_liveOpsService.LiveOpsToEndTime.ContainsKey(dynamicInAppOperation.Id))
    {
        OnShowNext(parent);
        return;
    }

    // ... rest of existing code
}
```

This fix is general-purpose and will prevent the same issue for any live op type, not just coupons.

**Option B (Simpler, complementary) — Downgrade the log in `ShopCouponService`:**

In `HandleLiveOpsSelectedAsync`, change `Debug.LogError` to `Debug.LogWarning` for the "not initialized" case, since this is a known race condition that is already gracefully handled:

```csharp
if (_dynamicInAppOperation == null)
{
    Debug.LogWarning($"[{ErrorCode.Coupons}] Coupons service is not initialized. " +
                   $"Passed live op id: {liveOpId} Time out: {timeLeft != null && timeLeft.Value <= TimeSpan.Zero}");
    return default;
}
```

**Recommendation**: Apply both Option A and Option B. Option A prevents the unnecessary call entirely; Option B ensures that if a similar race occurs through a different path, it doesn't pollute error logs.
