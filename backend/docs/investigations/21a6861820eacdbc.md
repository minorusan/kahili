# Investigation: ChallengeCriteriaPopup.Hide NRE
## TLDR
This is a high-volume null reference in the seasonal challenge criteria popup hide flow. The Hide async method dereferences a null `_cancellationTokenSource`, which can happen when Hide is called before Init/Show creates the token (e.g., popup active by default and user taps info quickly). Guarding or initializing the token (and/or ensuring the popup starts inactive) should stop the crash.
## Risk Assessment
Medium: it is a client-side exception with high frequency and user impact but likely recoverable UI state; still noisy and can break popup interactions.
## Release Branch
v3.34.6
## Release Version
3.34.6
## Error Summary
NullReferenceException in `ninja.LiveOpsChallenges.Popups.Logic.ChallengeCriteriaPopup+<Hide>d__16.MoveNext` during Hide delay/cancellation handling, triggered from popup show/hide flows in the seasonal challenge UI.
## Root Cause
`ChallengeCriteriaPopup.Hide()` uses `_cancellationTokenSource.Token` without checking for null. `_cancellationTokenSource` is only created in `Init()` and `Show()`. If Hide is invoked before either of those (e.g., the criteria popup is active by default and the user toggles it via the info button before `SetupView` calls `Init`, or any call path hides a popup that was never shown), the token source is null and the async state machine throws.
## Affected Code
- `Assets/Scripts/LiveOpsChallenges/Popups/Logic/ChallengeCriteriaPopup.cs` (`Hide`, `Show`, `Init`, `LateUpdate`, `OnDisable`)
- `Assets/Scripts/LiveOpsChallenges/Popups/Logic/SeasonalChallengeLogic_v2.cs` (`ShowHideChallengeCriterias`, `InstantiateChallengeCriterias`, `UpdateMilestones` call sites)
## Suggested Fix
- Initialize `_cancellationTokenSource` defensively (e.g., in `Awake`/`OnEnable`) and guard in `Hide()`:
  - If `_cancellationTokenSource == null`, either create a new one or use `CancellationToken.None` for the delay.
- Ensure the popup game object starts inactive in the prefab, and only becomes active via `Show()`, to avoid Hide being called before initialization.
- Optional: in `ShowHideChallengeCriterias`, if popup is active but `_cancellationTokenSource` is null, call `Show()` or `Init()` before Hide.
