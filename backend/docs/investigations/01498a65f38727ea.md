# Investigation: Album open pack collect click null reference
## TLDR
This is likely a moderate-impact client crash in the album pack open popup: clicking Collect can throw a NullReferenceException and break the flow. The most probable cause is `_albumController.Packs` being null (album live-op not initialized or cleared) when `OnCollectClick` tries to read pack data for analytics.

## Risk Assessment
Moderate: affects a core reward flow, but the pack is likely already claimed before the click, so the main risk is a broken UX/soft-lock and missing analytics rather than lost rewards.

## Release Branch
v3.34.6

## Release Version
3.34.6 (first seen), events show 3.34.0 as well

## Error Summary
NullReferenceException in `ninja.AlbumLogic.Popups.Logic.AlbumOpenPackLogic.OnCollectClick()` when the user presses Collect in the album pack popup. The stack is UI event-driven with no line info.

## Root Cause
`OnCollectClick` dereferences ` _albumController.Packs[_albumPackSku.PackId].PackClaimAnimation` without guarding for null `Packs`. `AlbumController.Packs` returns `AlbumModel.Packs`, which is null when live-op data is not initialized or has been cleared. If the popup stays open while album data resets (e.g., live-op ended, user changed, or model cleared), the click path will hit a null dictionary and throw.

## Affected Code
- `Assets/Scripts/Popups/ViewLogic/Album/AlbumOpenPackLogic.cs` (v3.34.6): `OnCollectClick()` and `Close()` use `_albumController.Packs[...]` without null checks.
- `Assets/Scripts/Album/AlbumModel.cs` (v3.34.6): `Packs` returns null when live-op data is not initialized.

## Suggested Fix
Guard access to pack data in `OnCollectClick` (and similarly in `Close`/`StartTracking`) by using `TryGetValue` only when `Packs` is non-null. If missing, log a warning and skip `PackType` (or use a safe default) so the collect flow and animation continue. Also consider disabling the Collect button until pack info is available, or ensure the popup is closed if the album model is cleared.
