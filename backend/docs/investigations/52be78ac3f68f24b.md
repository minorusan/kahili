# Investigation: Album open pack collect click NRE
## TLDR
This is a client-side NRE triggered when the user presses Collect in the album open-pack popup. The likely cause is `AlbumOpenPackLogic.OnCollectClick` dereferencing album data after the live-op model has been cleared (so `Packs` returns null), which can happen if the live op ends or model resets while the popup is still open.

## Risk Assessment
Medium: it crashes the Collect flow in a live-op popup, likely impacting reward collection UX and analytics; frequency is non-trivial (258 occurrences).

## Release Branch
v3.34.6

## Release Version
3.34.6 (first seen), events recorded from 3.34.0

## Error Summary
`NullReferenceException` in `ninja.AlbumLogic.Popups.Logic.AlbumOpenPackLogic.OnCollectClick`.

## Root Cause
`OnCollectClick` assumes album live-op data is still present and calls `_albumController.Packs[_albumPackSku.PackId].PackClaimAnimation` without null checks. `AlbumModel.Packs` returns null when `CurrentLiveOp` is cleared (e.g., live-op ended/reset while the popup is still open). This turns into an NRE at click time. There were no functional changes to `AlbumOpenPackLogic` between v3.34.4 and v3.34.6; this appears to be a latent null path exposed by runtime timing/LiveOps state.

## Affected Code
- `Assets/Scripts/Popups/ViewLogic/Album/AlbumOpenPackLogic.cs` (method `OnCollectClick`)
- `Assets/Scripts/Album/AlbumModel.cs` (`Packs` returns null when `CurrentLiveOp` is null)

## Suggested Fix
- Guard against nulls and missing packs in `OnCollectClick`:
  - Cache `PackClaimAnimation` (or the pack itself) in `ShowPopup` when the pack is known, and use the cached value in `OnCollectClick`.
  - Alternatively, use `var packs = _albumController?.Packs; if (packs != null && _albumPackSku != null && packs.TryGetValue(_albumPackSku.PackId, out var pack)) { ... } else { log + omit PackType }`.
- Consider making `AlbumModel.Packs` return an empty dictionary instead of null to reduce NRE surface, or avoid using `_albumController.Packs` in UI callbacks after LiveOp teardown.
