# Investigation: ShopCouponRedeemLogic CloseWithAnimation NRE
## TLDR
This is a high-volume but likely non-fatal client error triggered when the coupon redeem popup is closing, not during redemption itself. The null reference occurs in `CloseWithAnimation` when the popup view or its Animator is already destroyed or missing, so the close animation call crashes.
## Risk Assessment
Moderate: it can leave the popup in a bad visual state or spam errors, but it does not appear to block redemption flow directly since `_result` is set before the exception and the popup may already be closed externally.
## Release Branch
v3.34.6
## Release Version
3.34.6
## Error Summary
`NullReferenceException` in `Powerof.Popups.ViewLogic.ShopCouponRedeemLogic.CloseWithAnimation` called via `TimerComp.TimerProcessing`.
## Root Cause
`CloseWithAnimation` assumes `_popupView` and `_popupView.Animator` are valid. If the popup is closed/destroyed externally (e.g., popup manager replaces/forces close, or user closes while timer is still running), the timer callback can fire later and call `Close()`, which reaches `CloseWithAnimation` after the view has been destroyed or the Animator was never assigned. This results in a null reference before `ClosePopup()` can run.
## Affected Code
- `Assets/Scripts/Popups/ViewLogic/ShopCouponRedeemLogic.cs` (CloseWithAnimation uses `_popupView.Animator.SetBool` without null checks)
- `Assets/Scripts/Popups/Components/TimerComp.cs` (timer callback can fire after external close if not disposed)
- `Assets/Scripts/Popups/PopupBase.cs` (external close/destroy path does not dispose timer for this popup)
## Suggested Fix
1) Guard `CloseWithAnimation` against missing view/Animator: if `_popupView == null` or `_popupView.Animator == null`, skip animation and just `await ClosePopup()` (or return if already closed).
2) Ensure timer is disposed when the popup is closed externally: wire `closeAction` or override `ClosePopup` in `ShopCouponRedeemLogic` to call `_timer?.Dispose()` and null it, so timer callbacks cannot fire after destruction.
These changes prevent the NRE and still allow the popup to close cleanly.
