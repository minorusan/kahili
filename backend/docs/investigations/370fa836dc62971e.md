# Investigation: LiveOps popup chain loses parent operation
## TLDR
This error is noisy but likely low severity: it logs whenever a Personal Offers popup follows a Challenge popup because the parent operation has already been dequeued. The next popup tries to suspend a closed operation, the queue is empty, and the manager logs an error before falling back to enqueueing the popup. Users can see popups open without the intended suspend/resume flow; in edge cases this can cause overlaps or odd input blocking.
## Risk Assessment
Low crash risk but high volume; potential UX glitches around popup stacking (overlap, missing resume), especially in LiveOps-heavy sessions.
## Release Branch
v3.34.6
## Release Version
3.33.3 (first seen)
## Error Summary
`[ContinuousOperation] No ongoing operations. Args current <ninja.LiveOpsChallenges.Popups.Logic.ChallengePopupLogic> -> next <PlayPerfect.Popups.ViewLogic.PersonalOffers.PersonalOffersLogic>` raised from `ContinuousOperationsManager.SuspendOperation` during `PopupBase.OpenPopup` when a Personal Offers live-op is shown right after a Challenge popup.
## Root Cause
`ChallengesPresenter` passes the closing `ChallengePopupLogic` instance to `showNextCallback`. By the time the next live-op opens, the challenge popup has already been dequeued in `PopupBase.DestroyPopup`, so `ContinuousOperationsManager` has no ongoing operations to suspend and logs an error. The next popup is enqueued anyway, but the parent/child semantics are broken.
## Affected Code
- `Assets/Scripts/LiveOpsChallenges/ChallengesPresenter.cs`
- `Assets/Scripts/LiveOpsChallenges/Popups/Logic/ChallengePopupLogic.cs`
- `Assets/Scripts/Popups/PopupBase.cs`
- `Assets/Scripts/Managers/ContinuousOperationsManager.cs`
## Suggested Fix
In `ChallengesPresenter.OnPopupClose`, call `showNextCallback` with `parentOperation` (or `null`) instead of the just-closed `challengeOperation`, matching other presenters (e.g., `LevelStrikeChallengePresenter`, `QuestChallengePresenter`). Optionally add a defensive guard before `SuspendWith` to ignore parent ops that are no longer in the queue, but the primary fix is to stop passing the closed popup as the next parent.
