# Investigation: SKU sprite download failures
## TLDR
This is a high-volume error (14.8k occurrences) but likely a content/config issue rather than a code crash: SKU image URLs like `Sku/Fallback/sku_wheel_fallback.png` and `Sku/Album/sku_album_pack_*` cannot be fetched, so `SkuManager` logs an error after retries with no exception. The download path returns null when the CDN/local asset is missing or invalid, so the log reports `[No Exception]` and the UI likely falls back to missing/blank sprites.
## Risk Assessment
Medium: repeated failures degrade UX (missing SKU images) and generate noise; no evidence of app crash, but impacts monetization/UI surfaces.
## Release Branch
v3.34.6
## Release Version
3.34.6
## Error Summary
`SkuManager.LoadSpriteInternalAsync` logs "Unable to download SKU sprite" for multiple SKU paths (fallback and album pack sprites). The stack traces show only logged errors (no thrown exception), originating from `SkuManager` → `DownloadOrLocalPics.GetSpriteAsync` → `DownloadOrGetLocalPath`.
## Root Cause
`DownloadOrLocalPics.TryDownloadAndCacheAsync` returns null (non-success HTTP result or missing local asset) without throwing; `SkuManager` retries and then logs `[No Exception]`. This points to missing/incorrect SKU image paths or absent CDN/local assets for SKU sprites like `Sku/Fallback/sku_wheel_fallback.png`, `Sku/Fallback/sku_stamp_card_fallback.png`, and `Sku/Album/sku_album_pack_*`. Because the failure is a non-exceptional return, `SkuManager` has no exception to report.
## Affected Code
- `Assets/Scripts/SKU/SkuManager.cs` (LoadSpriteInternalAsync logs after retry, no exception) 
- `Assets/Scripts/Utils/DownloadOrLocalPics.cs` (TryDownloadAndCacheAsync returns null on non-success, GetSpriteAsync returns null)
## Suggested Fix
1. Verify and correct SKU image paths returned by the SKU provider(s) for album/fallback assets; ensure the exact case/paths exist in the CDN container `assets/` (e.g., `Sku/Album/sku_album_pack_1_wood.png`, `Sku/Fallback/sku_wheel_fallback.png`). Upload missing PNGs or update mappings to the correct existing asset names.
2. Add defensive fallback in `SkuManager.LoadSpriteInternalAsync`: if `GetSpriteAsync` returns null, try a known local fallback sprite (addressable) before logging an error, so UI doesn’t render empty.
3. Improve logging in `DownloadOrLocalPics.TryDownloadAndCacheAsync` to include HTTP status/result and URL; optionally bubble a detailed error back to `SkuManager` to replace `[No Exception]` with a meaningful cause.
